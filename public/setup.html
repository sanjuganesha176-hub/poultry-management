üìã Quick Start

Step 1: Install Dependencies

Open PowerShell in the project directory and run:

npm install

Step 2: Configure Firebase

You need to set up Firebase for authentication, database, and storage:

  1. Go to Firebase Console
  2. Create a new project
  3. Enable the following services:
    • Authentication (Phone + Email/Password)
    • Firestore Database
    • Storage
  4. Get your Firebase configuration from Project Settings

Step 3: Add Firebase Configuration

Copy the template file and add your Firebase credentials:

# Copy the template
copy public\js\firebase-config-template.js public\js\firebase-config.js

Then open public/js/firebase-config.js and replace with your actual Firebase config:

const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "YOUR_PROJECT.firebaseapp.com",
    projectId: "YOUR_PROJECT_ID",
    storageBucket: "YOUR_PROJECT.appspot.com",
    messagingSenderId: "YOUR_SENDER_ID",
    appId: "YOUR_APP_ID"
};
‚ö†Ô∏è Important: The file firebase-config.js is in .gitignore to protect your credentials. Never commit your actual Firebase keys to version control!

Step 4: Configure Firebase Security Rules

Set up security rules in Firebase Console:

  • Firestore Rules: Go to Firestore ‚Üí Rules tab
  • Storage Rules: Go to Storage ‚Üí Rules tab

See FIREBASE_SETUP.md for detailed security rules.

Step 5: Start the Server

Run the development server:

npm start

The server will start on http://localhost:3000

üöÄ Production Deployment

Before deploying to production:
  1. Update Firebase security rules (remove test mode)
  2. Enable Firebase App Check
  3. Use environment variables for sensitive data
  4. Set up proper password hashing
  5. Add HTTPS support
  6. Configure authorized domains in Firebase
